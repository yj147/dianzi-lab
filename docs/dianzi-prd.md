## 执行摘要

Bambi Lab Idea 是一个软件开发服务平台，允许用户提交创意点子，由管理员审核并推进开发流程。该平台解决了"用户有想法但缺乏实现能力"的痛点，通过众包模式收集创意，由团队筛选并实现优质点子。

本项目为课程作业，需满足7项核心功能要求：首页展示、双端登录、点子提交、后台管理、进度追踪。采用 Next.js + Supabase 技术栈，玻璃态设计风格，部署于 Vercel。

---

## 问题陈述

**当前状况**：用户有创意想法但缺乏技术能力实现，或不知道如何将想法产品化。

**解决方案**：提供一个点子提交与实现平台，用户提交创意，管理员审核后推进开发，完成后在首页展示。

**业务影响**：完成课程作业7项功能要求，构建可展示的全栈项目。

## 用户画像

### 主要用户：普通用户

- **角色**：创意提交者
- **目标**：提交自己的点子，追踪进度，查看已实现的工具
- **痛点**：有想法但无法实现，不知道进度
- **技术水平**：初级

### 次要用户：管理员

- **角色**：点子审核与管理者
- **目标**：审核点子、管理状态流转、管理用户
- **痛点**：需要高效管理大量点子
- **技术水平**：中级

## 用户故事与验收标准

### US-1: 首页展示已完成点子

**作为** 访客或用户
**我想要** 在首页看到所有已完成的点子/工具
**以便** 了解平台能力和浏览可用工具

**验收标准**：

- [ ] 首页包含 Hero 区域，展示平台介绍和核心 CTA
- [ ] Hero 下方以卡片网格展示所有"已完成"状态的点子
- [ ] 每张卡片显示：标题、描述摘要、标签
- [ ] 支持响应式布局（移动端单列，桌面端多列）
- [ ] 空状态显示引导文案

### US-2: 用户注册与登录

**作为** 访客
**我想要** 通过邮箱密码注册和登录
**以便** 获得提交点子的权限

**验收标准**：

- [ ] 提供注册页面：邮箱、密码、确认密码
- [ ] 提供登录页面：邮箱、密码
- [ ] 邮箱格式验证，密码最少6位
- [ ] 登录成功后跳转至用户中心
- [ ] 登录状态持久化（刷新不丢失）

### US-3: 管理员登录

**作为** 管理员
**我想要** 使用预设账号登录管理后台
**以便** 管理平台内容

**验收标准**：

- [ ] 管理员使用独立登录入口或相同登录页
- [ ] 管理员账号可通过数据库种子数据预设（生产环境密码通过环境变量注入，禁止默认口令）
- [ ] 登录后自动识别角色，跳转至管理后台
- [ ] 普通用户无法访问管理后台

### US-4: 用户提交点子

**作为** 登录用户
**我想要** 提交我的创意点子
**以便** 有机会被实现

**验收标准**：

- [ ] 提交流程分三步：点子信息 → 创意评估 → 结果展示
- [ ] 第一步：填写点子表单（标题、描述、标签）
- [ ] 第二步：填写 9 维度评估表单（目标用户、渠道、市场、技术、预算、商业模式、团队、风险、流量）
- [ ] 标题最大50字符，描述最大1000字符
- [ ] 评分 < 50 时阻止提交，显示警告和反馈建议
- [ ] 评分 >= 50 时正常提交，关联评估结果，跳转结果页
- [ ] 结果页展示雷达图、评分、反馈建议
- [ ] 提交成功显示反馈提示
- [ ] 未登录用户点击提交引导至登录

### US-5: 管理后台管理点子

**作为** 管理员
**我想要** 查看和管理所有点子
**以便** 推进点子开发流程

**验收标准**：

- [ ] 点子列表支持按状态筛选：全部/待审核/已采纳/开发中/已完成
- [ ] 可更改点子状态：待审核 → 已采纳 → 开发中 → 已完成
- [ ] 点子详情页支持维护报价与支付状态（待支付/已支付/已退款）并记录付款时间
- [ ] 可将点子移至垃圾箱（软删除）
- [ ] 垃圾箱可恢复或永久删除
- [ ] 提供统计看板：各状态点子数量
- [ ] 提供用户管理：查看用户列表

### US-6: 已完成点子首页展示

**作为** 任何访问者
**我想要** 在首页看到已完成的点子
**以便** 了解平台成果

**验收标准**：

- [ ] 点子状态变为"已完成"后自动出现在首页
- [ ] 状态从"已完成"变更后从首页移除
- [ ] 首页按完成时间倒序排列

### US-7: 用户查看点子进度

**作为** 提交点子的用户
**我想要** 在用户中心查看我的点子进度
**以便** 了解处理情况

**验收标准**：

- [ ] 用户中心显示"我的点子"列表
- [ ] 每个点子显示当前状态文字（待审核/已采纳/开发中/已完成）
- [ ] 状态使用不同颜色区分
- [ ] 按提交时间倒序排列

## 功能需求

### 核心功能

**F1: 首页**

- Hero 区域：标题、副标题、两个 CTA 按钮（提交点子、浏览工具）
- 工具展示区：卡片网格布局展示已完成点子
- 导航栏：Logo、首页、提交点子、登录/用户头像

**F2: 认证系统**

- 用户注册：邮箱 + 密码
- 用户登录：邮箱 + 密码
- 管理员登录：复用登录页，角色区分
- 会话管理：JWT 或 Supabase Auth

**F3: 点子提交**

- 表单：标题、描述、标签选择
- 预设标签：工具、效率、娱乐、学习、其他
- 提交验证与反馈

**F4: 用户中心**

- 我的点子列表
- 状态展示（带颜色标识）
- 退出登录

**F5: 管理后台**

- 仪表板：统计数据概览
- 点子管理：列表 + 状态筛选 + 状态变更
- 点子详情：报价与支付状态管理（含付款时间）
- 用户管理：用户列表查看
- 垃圾箱：软删除点子管理

### 不在范围内

- 点子编辑/删除（用户端）
- 邮件通知
- 评论/投票功能
- 附件上传
- 多语言支持

## 技术约束

### 技术栈

- **前端框架**：Next.js 14 (App Router)
- **样式方案**：Tailwind CSS
- **ORM**：Prisma
- **数据库**：Supabase PostgreSQL
- **认证**：Supabase Auth 或 NextAuth.js
- **部署**：Vercel

### 性能要求

- 首页首次加载 < 3秒
- API 响应 < 500ms

### 安全要求

- 密码哈希存储
- API 路由鉴权
- 管理后台访问控制
- XSS/CSRF 防护

### 兼容性

- 现代浏览器（Chrome、Firefox、Safari、Edge）
- 响应式适配（移动端 375px+，桌面端 1024px+）

User
├── id: UUID
├── email: String (unique)
├── password: String (hashed)
├── role: Enum (USER, ADMIN)
├── createdAt: DateTime
└── ideas: Idea[]

Idea
├── id: UUID
├── title: String
├── description: Text
├── status: Enum (PENDING, APPROVED, IN_PROGRESS, COMPLETED)

├── createdAt: DateTime
